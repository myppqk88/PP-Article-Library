设置

安装：brew install codex（macOS） 或下载 Codex.app；首次使用先在终端跑一次 codex login。
路径示例：macOS /Applications/Codex.app/Contents/Resources/codex；Windows codex.cmd（在 PATH 中可省略全路径）。

安装：npm install -g @anthropic-ai/claude-code；首次使用先在终端运行 claude login（推荐，免管理 key）。
或者：把 ANTHROPIC_API_KEY 填到上面，会写入 .env，CLI 会自动读取。
路径示例：macOS/Linux claude；Windows claude.cmd（在 PATH 中可省略全路径）。
模型示例：claude-sonnet-4-5、claude-opus-4-1、claude-haiku-4-5。

本地部署，零成本。Mac 上 brew services start ollama 启动；Windows 用官方安装器。
如果在另一台机器跑了 Ollama，填那台机器的 LAN 地址，例如 http://192.168.1.100:11434。

· DeepSeek：https://api.deepseek.com + deepseek-chat
· Qwen：https://dashscope.aliyuncs.com/compatible-mode/v1 + qwen-plus
· OpenAI：https://api.openai.com/v1 + gpt-4o-mini

翻译快捷键

点上面输入框，按一组键（如 Alt+T / Ctrl+Q）即可设为「翻译」快捷键。
选中网页里的文字后按快捷键直接翻译；PDF 里的文字仍需先 Ctrl+C。留空 = 不启用快捷键。

扫描件 OCR fallback：当 PDF 抽出的文本太少（少于「触发阈值」字符）或被启发式判定为扫描件时，自动用 OCR 引擎识别每页文字。
结果缓存在 library/text/{hash}.ocr.txt，删除该文件可强制重新 OCR。

云 OCR 配置（仅当引擎=云 OCR 时）

云 OCR 走任意 OpenAI 兼容的视觉接口。常用：
· 阿里通义千问 VL：https://dashscope.aliyuncs.com/compatible-mode/v1 + qwen-vl-plus
· 火山豆包 OCR：参 火山文档
· OpenAI GPT-4o-mini：https://api.openai.com/v1 + gpt-4o-mini

视觉模型：当 AI 问答附带 PDF 页面图片时（在 AI 面板填页码 + 勾选「加图」），调用走的不是上面的主模型，而是这里配置的视觉 provider。
主模型保持文本（DeepSeek 之类便宜快），视觉单独走 → 节省 token + 主模型不必支持视觉。

在 阿里云 DashScope 控制台 拿 API key。模型对比：
· qwen-vl-plus：~¥0.008/图，速度快，日常文献图够用
· qwen-vl-max：~¥0.02/图，精度更高，复杂表格/流程图更稳
如果你已在主模型用过 Qwen，环境变量复用就行。

· gpt-4o-mini：~$0.001/图，便宜，理解力够用
· gpt-4o：~$0.005/图，高复杂度图表更准
如果有国内代理，把 Base URL 改成代理地址。

第一次使用配置向导

把主 AI、翻译、可选服务一次填完；之后也可以从「更多操作 → 配置向导」重新打开。

先选一个主 AI

主 AI 用来整理新文献、生成笔记、帮我阅读、帮我引用。新用户最省心的是 DeepSeek 或 Qwen：填一个 API key 就能用。

再选翻译方式

划词翻译可以走本地 Ollama（有些用户口头叫“欧拉”），也可以直接复用上一步的 API。没装本地模型时，选 API 最省事。

复用主 API 时，不需要再填一份 key。翻译会使用轻量模型，适合多数用户先跑起来。

可选功能，先跳过也没关系

这些不是启动必需项。EasyScholar 用来查期刊等级；视觉模型用于带页面截图问 AI；OCR 用于扫描版 PDF。

保存后就可以开始用了

这些配置会写入本地 `config/settings.yaml` 和 `.env`。公开 GitHub 仓库不会包含你的 key、PDF、笔记或 citation 文件。
